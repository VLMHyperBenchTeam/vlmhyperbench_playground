# Задача 9: Реализация Reporting Engine ✅ ВЫПОЛНЕНО

## Цель
Создать модуль генерации отчетов, который агрегирует результаты оценки, метрики производительности и метаданные эксперимента в человекочитаемый формат (Markdown/HTML) и структурированный JSON для аналитики.

## Контекст
См. [Concept](../docs_site/docs/architecture/concept.md).
После этапа оценки (`MetricEvaluator`) мы получаем сырые CSV файлы с метриками. Нам нужно превратить их в итоговый артефакт, который можно прочитать или использовать для сравнения моделей.

## Подзадачи

1.  **Report Generator Module**:
    *   Создать пакет `packages/reporting`.
    *   Класс `ReportGenerator`, принимающий на вход пути к `metrics.csv`, `answers.csv` и конфигу эксперимента.

2.  **Шаблонизация (Jinja2)**:
    *   Разработать шаблоны отчетов (`templates/report.md.j2`).
    *   Секции отчета:
        *   **Summary**: Сводная таблица (Model Name, Accuracy, Latency, VRAM).
        *   **Detailed Metrics**: Разбивка по категориям документов.
        *   **Performance**: Графики (текстовые или ссылки на изображения) потребления ресурсов.
        *   **Errors**: Топ-5 самых частых ошибок (с примерами).

3.  **Агрегация данных**:
    *   Логика вычисления средних значений, перцентилей (p95, p99) для Latency.
    *   Сравнение с Baseline (если указан в конфиге).

4.  **Экспорт**:
    *   Генерация `final_report.md` (для чтения в Git/Obsidian).
    *   Генерация `report_data.json` (для импорта в Web UI Dashboard).

## Ожидаемый результат
*   [x] Пакет `reporting`, генерирующий красивые Markdown отчеты.
*   [x] Отчет содержит не только цифры качества (Accuracy), но и технические метрики (Speed/Memory).
*   [x] JSON-артефакт готов для визуализации в дашборде.

## Реализация
1.  **Пакет `packages/reporting`**: Создан с использованием `src` layout и `uv.build` backend.
2.  **Модуль `ReportGenerator`**: Реализован в `generator.py`. Использует `pandas` для агрегации метрик (mean, median, p95, p99).
3.  **Шаблоны**: Использование Jinja2 для генерации Markdown отчетов на основе шаблона `report.md.j2`.
4.  **Тестирование**: Добавлены тесты в `tests/test_generator.py`, подтверждающие корректность работы через `uv run pytest`.
