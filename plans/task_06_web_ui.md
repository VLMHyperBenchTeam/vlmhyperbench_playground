# Задача 6: Реализация Web UI (React)

## Цель
Создать пользовательский интерфейс для управления экспериментами, визуализации прогресса и анализа результатов.

## Контекст
См. [ADR-008: React Dashboard](../docs_site/docs/architecture/adr/008-react-dashboard-with-plotly.md) и [Specification](../docs_site/docs/architecture/specification.md).
UI должен быть отзывчивым, поддерживать real-time обновления и предоставлять удобные инструменты для работы с графиками.

**Дизайн-код**: Интерфейс должен напоминать современную IDE (VS Code style) — темная тема по умолчанию, компактные панели, моноширинные шрифты для логов и данных.

## Подзадачи

1.  **Инициализация проекта**:
    *   Создать директорию `web_ui/`.
    *   Инициализировать проект на React + Vite + TypeScript.
    *   Настроить Tailwind CSS с палитрой цветов, имитирующей VS Code Dark Theme.
    *   Подключить Shadcn UI (или аналогичные компоненты) и настроить их стилизацию под "IDE-look".

2.  **Landing & Auth (Cyberpunk/IDE Style)**:
    *   **Landing Page**: Минималистичная, но стильная страница.
        *   Акцент на логотип (SVG).
        *   Стиль: Смесь "Cyberpunk" и "Professional IDE" (темные тона, неоновые акценты, но без визуального шума).
        *   Краткое описание возможностей (без лишнего маркетинга).
    *   **Auth Page**: Форма входа в том же стиле.

3.  **Основные страницы (Protected Routes)**:
    *   **Dashboard**: Список активных и завершенных экспериментов с краткими статусами.
    *   **New Experiment**: Форма создания эксперимента (загрузка CSV/JSON конфига, выбор пресетов).
    *   **Experiment Detail**:
        *   Прогресс-бары этапов.
        *   Терминал с логами (Streaming) — стилизация под встроенный терминал VS Code.
        *   Вкладка "Metrics" с графиками Plotly.

4.  **Вкладка Monitoring (Resource Watchdog)**:
    *   Визуализация состояния воркеров (Docker контейнеров).
    *   Real-time графики потребления ресурсов (GPU Util, VRAM, CPU, RAM).
    *   Отображение статуса подключения к API Wrapper'ам.

5.  **Интеграция с Backend**:
    *   Настроить API Client (Axios/TanStack Query).
    *   Реализовать WebSocket-клиент для получения обновлений статуса и метрик ресурсов.

6.  **Визуализация (Plotly)**:
    *   Интегрировать `react-plotly.js`.
    *   Реализовать компоненты для отображения графиков, получаемых от `/analytics/{id}`.
    *   Настроить темную тему для самих графиков Plotly.

## Ожидаемый результат
*   Web-приложение, доступное по адресу `http://localhost:3000`.
*   Стильный лендинг и страница авторизации (Cyberpunk/IDE aesthetic).
*   Темная тема "из коробки", профессиональный "инженерный" интерфейс внутри.
*   Отдельная вкладка для мониторинга здоровья и ресурсов воркеров.
