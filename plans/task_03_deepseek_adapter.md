# Задача 3: Адаптер для DeepSeek-OCR

## Цель
Обеспечить поддержку модели DeepSeek-OCR, которая требует специфических параметров генерации и обработки входных данных.

## Контекст
DeepSeek-OCR может требовать особых настроек, таких как:
*   Ограничение области поиска (bounding boxes).
*   Формат вывода (Markdown, JSON, Plain Text).
*   Специфичные токены остановки.

## Подзадачи

1.  **Исследование требований модели**:
    *   Изучить документацию DeepSeek-OCR (или примеры использования).
    *   Определить список необходимых параметров (`model_params`).

2.  **Реализация адаптера (в рамках API Wrapper)**:
    *   Создать `DeepSeekAdapter` (наследуется от `AbstractBackend` или расширяет `vLLMAdapter`).
    *   Реализовать маппинг параметров из запроса `/v1/chat/completions` в аргументы движка генерации.

3.  **Конфигурация**:
    *   Добавить пример конфигурации в `vlm_base.csv` (или аналогичный реестр) с указанием класса адаптера.

4.  **Тестирование**:
    *   Локальный запуск на тестовом изображении.
    *   Проверка корректности передачи параметров.

## Ожидаемый результат
*   Модель DeepSeek-OCR успешно запускается через API Wrapper.
*   Параметры, переданные в `model_params`, влияют на генерацию.
