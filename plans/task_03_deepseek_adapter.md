## Результаты
1.  **Выделенный пакет**: Создан пакет [`packages/deepseek_adapter`](../../packages/deepseek_adapter), наследующий базовую логику vLLM.

3.  **Интеграция с Реестрами**: Спроектирована схема `FrameworkDefinition` для автоматического подбора этого адаптера при указании моделей семейства DeepSeek.
4.  **Тестирование**: Реализованы unit-тесты, проверяющие корректность модификации параметров запроса.

## Цель
Обеспечить поддержку модели DeepSeek-OCR, которая требует специфических параметров генерации и обработки входных данных.

## Контекст
DeepSeek-OCR может требовать особых настроек, таких как:
*   Ограничение области поиска (bounding boxes).
*   Формат вывода (Markdown, JSON, Plain Text).
*   Специфичные токены остановки.

## Подзадачи

1.  **Исследование требований модели**: ✅ ВЫПОЛНЕНО
    *   Определены специфические стоп-токены для корректного завершения генерации DeepSeek-OCR.

2.  **Реализация адаптера**: ✅ ВЫПОЛНЕНО
    *   Создан пакет `packages/deepseek_adapter`.
    *   `DeepSeekAdapter` наследуется от `vLLMAdapter` и автоматически инжектирует стоп-токены в параметры генерации.

    *   Спроектирована схема `FrameworkDefinition` для автоматического маппинга моделей DeepSeek на данный адаптер.

4.  **Тестирование**: ✅ ВЫПОЛНЕНО
    *   Реализованы unit-тесты, подтверждающие корректную модификацию запросов адаптером.
