# Задача 14: Рефакторинг Dataset Iterator

## Цель
Обеспечить поддержку расширяемости датасетов. Пользователь должен иметь возможность легко добавить поддержку своего формата датасета, создав новый класс итератора.

## Контекст
Текущая реализация `dataset_iterator` (предположительно) жестко привязана к определенным форматам. Мы хотим, чтобы `RunTask` или `MLTask` могли указывать тип датасета, а система динамически подгружала нужный итератор.

## Подзадачи

1.  **Dataset Interface**:
    *   Определить абстрактный базовый класс `DatasetIterator`.
    *   Методы: `__iter__`, `__len__`, `get_item(idx)`.
    *   Возвращаемый тип: унифицированный объект `DatasetItem` (image_path, text, metadata).

2.  **Dataset Registry**:
    *   Создать реестр итераторов (аналогично Metric Registry).
    *   Декоратор `@register_dataset(name)` для автоматической регистрации.

3.  **Refactor Fabrics**:
    *   Обновить `IteratorFabric` (или заменить на Registry), чтобы он создавал итератор по имени, указанному в конфиге задачи (`dataset_type`).

4.  **Custom Dataset Support**:
    *   Пользователь может положить свой скрипт с классом итератора в `packages/custom_datasets/` и зарегистрировать его.
    *   Оркестратор должен уметь подгружать эти пользовательские модули.

## Ожидаемый результат
*   Пакет `dataset_iterator` поддерживает плагинную архитектуру.
*   Добавление нового формата датасета не требует изменения ядра фреймворка.
